ROUTE ANALYSIS - CricAlgo API Routes
=====================================

MAIN ROUTER MOUNTING (app/main.py):
- health_router: /api/v1
- webhooks_router: /api/v1  
- auth_router: /api/v1
- wallet_router: /api/v1
- contest_router: /api/v1
- admin_router: /api/v1
- admin_contest_router: /api/v1/admin
- debug_router: /api/v1/debug

CONTEST ROUTES (app/api/v1/contest.py):
- POST /{contest_id}/join - Join contest endpoint
- POST /admin/contest - Create contest (admin)
- POST /admin/{contest_id}/settle - Settle contest (admin)
- GET / - Get contests list
- GET /{contest_id} - Get contest details

ISSUE IDENTIFIED:
=================
The contest join route is defined as:
  @router.post("/{contest_id}/join")

With contest_router mounted at "/api/v1", the full path becomes:
  /api/v1/{contest_id}/join

But the smoke test is calling:
  /api/v1/contest/{contest_id}/join

The smoke test is using "contest" (singular) in the path, but the router doesn't have a "contest" prefix.

SMOKE TEST URL (line 514-515):
  f"{self.base_url}/api/v1/contest/{self.contest_id}/join"

ACTUAL ROUTE:
  /api/v1/{contest_id}/join

SOLUTION:
Either:
1. Change the route to include "contest" prefix: @router.post("/contest/{contest_id}/join")
2. Change smoke test to use: /api/v1/{contest_id}/join

Option 1 is preferred as it's more RESTful and matches the pattern used by other endpoints.
